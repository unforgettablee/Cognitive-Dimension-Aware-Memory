# 消融实验设置

## 一、实验范式

全部使用 **Sequential** 范式。每轮一个任务，跑完后从通过的 trial 中提取认知记忆（4×4 矩阵 + 传统三层）到共享池，下一轮注入 instruction。

**任务选取**：SWE-bench Verified 共 500 任务。E1/E2/E7 跑全部 500 任务，其余消融使用 `num_tasks: 100` + `random_seed: 42` 随机固定采样 **同一组 100 任务**，保证跨实验可比。

**结果组织**：所有 trial 合并到 `${jobs_dir}/${启动时间戳}/` 下，每个 run 生成 `sequential_summary.json`（含 `selected_tasks`），用于 `compare_ablation.py` 逐任务对齐。

## 三、4×4 认知矩阵消融（4 个）

> 验证 COGENT 的核心创新 — 4 个认知维度各自是否必要。使用 **leave-one-out** 设计：从完整 4×4 矩阵中去掉一个维度，剩余三维 + 完整 LLM 管线运行。

| ID | 实验 | 检索池 | 核心问题 |
|:--:|------|--------|----------|
| E8 | 去掉 causal | contrastive + strategic + environment × 4 层 | causal（因果推理）是否冗余？ |
| E9 | 去掉 contrastive | causal + strategic + environment × 4 层 | contrastive（反模式）是否有独立价值？ |
| E10 | 去掉 strategic | causal + contrastive + environment × 4 层 | strategic（方法论策略）是否冗余？ |
| E11 | 去掉 environment | causal + contrastive + strategic × 4 层 | environment（环境知识）是否冗余？ |

**预期**：去掉任一维度后 pass rate 下降。若某维度去掉后无显著变化，说明该维度可精简。

## 五、检索管线健壮性检验（4 个）

> 验证 pipeline 中的设计选择是否必要。

| ID | 实验 | 变量 | 核心问题 |
|:--:|------|------|----------|
| E15 | 随机记忆 | 从 top-20 语义候选中随机抽 3 条 | COGENT 检索是否显著优于从相关候选中的随机选择？ |
| E16 | LLM 直接 top-K | 不分 4×4 维度，LLM 从 20 候选直接挑 3 | 认知维度划分是否必要？ |
| E17 | 固定阈值 | `score_threshold_std: 0.0`，用全局固定阈值 | 动态阈值（μ ± kσ）是否必要？ |
| E18 | 去掉 insight 嵌入 | `attach_insights: false`，不将 abstract insight 附着到具体记忆 | insight layering（抽象原则嵌入）是否提供独立增量？ |

## 九、结果对比

E1、E2、E7 是 500 任务全量，其他消融是 100 任务。`compare_ablation.py` 会自动从全量中提取相同的 100 任务进行对齐。

```bash
python harbor/compare_ablation.py \
  --baseline jobs/sequential/e7_no_memory/<ts>/sequential_summary.json \
  --experiments \
    jobs/sequential/e6_mtl_original/<ts>/sequential_summary.json \
    jobs/sequential/e5_embedding/<ts>/sequential_summary.json \
    jobs/sequential/e4_no_synergy/<ts>/sequential_summary.json \
    jobs/sequential/e3_no_rerank/<ts>/sequential_summary.json \
    jobs/sequential/e8_no_causal/<ts>/sequential_summary.json \
    jobs/sequential/e9_no_contrastive/<ts>/sequential_summary.json \
    jobs/sequential/e10_no_strategic/<ts>/sequential_summary.json \
    jobs/sequential/e11_no_environment/<ts>/sequential_summary.json \
    jobs/sequential/e12_no_trajectory/<ts>/sequential_summary.json \
    jobs/sequential/e13_no_workflow/<ts>/sequential_summary.json \
    jobs/sequential/e14_no_summary/<ts>/sequential_summary.json \
    jobs/sequential/e15_random_memory/<ts>/sequential_summary.json \
    jobs/sequential/e17_fixed_threshold/<ts>/sequential_summary.json \
    jobs/sequential/e18_no_insight/<ts>/sequential_summary.json \
    jobs/sequential/e1_full/<ts>/sequential_summary.json \
    jobs/sequential/e2_full/<ts>/sequential_summary.json \
  --output-dir comparisons/
```

输出每个实验一份 `comparison_<name>.json`：整体 & matched pass rate、逐任务 PASS/FAIL 对比、Δ 绝对值和相对提升
