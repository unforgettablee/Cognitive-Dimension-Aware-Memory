# 消融实验设置

## 一、实验范式

全部使用 **Sequential** 范式。每轮一个任务，跑完后从通过的 trial 中提取认知记忆（4×4 矩阵 + 传统三层）到共享池，下一轮注入 instruction。

**任务选取**：SWE-bench Verified 共 500 任务。所有实验使用 `num_tasks: 100` + `random_seed: 42` 随机固定采样 **同一组 100 任务**，保证跨实验可比。E6（无记忆基线）额外跑全部 500 任务提供稳健参照。

**结果组织**：所有 trial 合并到 `${jobs_dir}/${启动时间戳}/` 下，每个 run 生成 `sequential_summary.json`（含 `selected_tasks`），用于 `compare_ablation.py` 逐任务对齐。

## 三、4×4 认知矩阵消融（4 个）

> 验证 COGENT 的核心创新 — 4 个认知维度各自是否必要。使用 **leave-one-out** 设计：从完整 4×4 矩阵中去掉一个维度，剩余三维 + 完整 LLM 管线运行。

| ID | 实验 | 检索池 | 核心问题 |
|:--:|------|--------|----------|
| E7 | 去掉 causal | contrastive + strategic + environment × 3 层 | causal（因果推理）是否冗余？|
| E8 | 去掉 contrastive | causal + strategic + environment × 3 层 | contrastive（反模式）是否有独立价值？|
| E9 | 去掉 strategic | causal + contrastive + environment × 3 层 | strategic（方法论策略）是否冗余？|
| E10 | 去掉 environment | causal + contrastive + strategic × 3 层 | environment（环境知识）是否冗余？|

**预期**：去掉任一维度后 pass rate 下降。若某维度去掉后无显著变化，说明该维度可精简。

## 六、记忆池规模 & 跨仓库（搁置）

> 以下实验在 Sequential 范式下存在设计困难，暂不纳入当前消融计划。

**记忆池规模分析**（E18-E21）：在 Sequential 范式下，记忆池是逐步累积的，"限制池为 N 个任务"意味着前 N 个任务充当池构建阶段，后续任务才是真正的评估对象。但这引入了任务顺序的混杂变量 — 前 N 个任务恰好是哪些仓库/难度完全由 random seed 决定。更适合的设计是**两阶段**：先独立建池，再用固定池评估所有任务。

**跨仓库迁移**（E22-E24）：需要先跑仓库 A 的全量任务建池，再切到仓库 B 评估。

## 十、结果对比

E1 和 E6 是 500 任务全量，其他消融是 100 任务。`compare_ablation.py` 会自动从 E1/E6 的 500 任务中提取相同的 100 任务进行对齐。

```bash
python harbor/compare_ablation.py \
  --baseline jobs/sequential/e6_no_memory/<ts>/sequential_summary.json \
  --experiments \
    jobs/sequential/e5_mtl_original/<ts>/sequential_summary.json \
    jobs/sequential/e4_embedding/<ts>/sequential_summary.json \
    jobs/sequential/e3_no_synergy/<ts>/sequential_summary.json \
    jobs/sequential/e2_no_rerank/<ts>/sequential_summary.json \
    jobs/sequential/e16_fixed_threshold/<ts>/sequential_summary.json \
    jobs/sequential/e17_no_insight/<ts>/sequential_summary.json \
    jobs/sequential/e1_full/<ts>/sequential_summary.json \
  --output-dir comparisons/
```

输出每个实验一份 `comparison_<name>.json`：整体 & matched pass rate、逐任务 PASS/FAIL 对比、Δ 绝对值和相对提升
